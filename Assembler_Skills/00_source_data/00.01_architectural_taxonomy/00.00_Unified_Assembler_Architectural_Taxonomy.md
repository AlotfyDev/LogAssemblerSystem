# 4. L-003 — Runtime / State Entities

## 4.1 Layer Definition

Runtime/state entities describe the internal runtime structures over which the Assembler’s queue, slot, dispatch, moderation, and waiting-list protocols operate.

## 4.2 Governing Correction

The corrected runtime rule is:

> **Queue = Container**

A queue is not inside a separate container.
A queue is itself a container holding slots.

# 13. Canonical Unified Statement

## 13.1

The Assembler Subsystem taxonomy is now organized as a unified, layered architecture in which external actors, internal components, runtime state structures, carrier objects, typed contract families, and constitutional doctrines are separately classified but connected through stable relation IDs.

## 13.2

The Assembler receives content through typed ingress contracts, realizes authoritative Log Envelopes, applies metadata and timestamp preparation, consumes generic Policy Objects through an AssemblerPolicyBundle and PolicyAssigner, hosts component-specific policy mechanisms inside internal components, manages queue-as-container runtime structures through round managers, waiting lists, moderators, and queue bundle agents, and terminates its authority at dispatcher-mediated handoff to the Log Level Records Registry.

## 13.3

The unified graph must preserve the following laws:

1. Dispatcher is the final assembler-side boundary.
2. Policy Object is a generic assignable doctrine object.
3. Components host mechanisms; policies govern them.
4. Contracts are typed/template laws, not runtime COM interfaces.
5. Log Envelope is the authoritative runtime transport unit.
6. Queue is Container.
7. Round Manager waiting lists carry queue-container identifiers, not envelopes.
