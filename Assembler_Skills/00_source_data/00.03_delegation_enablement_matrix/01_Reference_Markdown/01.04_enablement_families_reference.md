# 1. Enablement Concept

## 1.1 Definition

### 1.1.1

An **enablement** is the architectural capability, surface, or mechanism required for a delegated entity to carry a responsibility.

### 1.1.2

Enablement answers:

```text id="0orf0l"
What must this entity be able to do?
```

This definition is aligned with the assignment principles document, which states that enablement is the required capability, surface, or mechanism that allows a delegated entity to carry a responsibility. 

---

## 1.2 Enablement Is Not Implementation

### 1.2.1

Enablement is not:

1. final code,
2. final method name,
3. final class interface,
4. final package boundary,
5. final runtime algorithm,
6. final PostgreSQL schema.

### 1.2.2

Enablement may later derive:

1. capability candidates,
2. interface-like template requirements,
3. validation rules,
4. lifecycle guards,
5. runtime orchestration capabilities,
6. observability/reporting surfaces.

---

## 1.3 Enablement vs Responsibility

### 1.3.1

A responsibility states what must be achieved.

### 1.3.2

An enablement states what capability surface is required to achieve it.

### 1.3.3

Example:

| Responsibility                                                                  | Enablement                                   |
| ------------------------------------------------------------------------------- | -------------------------------------------- |
| `RSP-CAN-009` Assign Policy Objects to policy-consuming components              | `EN-FAM-001` Assignment Surface Enablement   |
| `RSP-CAN-027` Assemble content, metadata, and timestamp into Log Level Envelope | `EN-FAM-003`, `EN-FAM-009`                   |
| `RSP-CAN-025` Update delivery-cycle flags after delivery round completion       | `EN-FAM-006` Feedback / Reporting Enablement |
| `RSP-CAN-044` Maintain writing round waiting list                               | `EN-FAM-004` Queue Runtime Enablement        |

---

## 1.4 Enablement vs Governed Object

### 1.4.1

A governed object is affected by a responsibility.

### 1.4.2

An enablement is the capability needed by the entity carrying the responsibility.

### 1.4.3

Example:

```text id="d5z98k"
Dispatcher updates Delivery-Cycle Flag Set.
```

In this example:

1. `Dispatcher` is the delegated entity.
2. `Delivery-Cycle Flag Set` is the governed object.
3. `EN-FAM-006` is the required enablement family.

### 3.4.1

`EN-FAM-001` requires:

1. policy bundle intake compatibility,
2. Policy Object identity preservation,
3. assignment-readiness tracking,
4. target component assignment surface,
5. safe-point replacement coordination,
6. traceable assignment state.

---

## 3.5 Obligations Preserved

| Obligation ID | Obligation                                                                       |
