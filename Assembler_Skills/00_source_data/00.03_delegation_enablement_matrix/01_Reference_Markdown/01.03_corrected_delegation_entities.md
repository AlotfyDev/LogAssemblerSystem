## 1.1

The entity corrections in this document are derived from four source layers.

The contract layer expands raw Batch 2 contracts into canonical contract families, including ingress, policy, metadata, operational, registry, envelope, and exclusion families. 

## 1.3 Taxonomy Source

The unified taxonomy defines the original internal entity layer, runtime/state layer, boundary/carrier object layer, doctrine rules, and relation families.

The corrected delegation matrix explicitly introduces:

The assignment principles require the Assembler Subsystem to remain accountable while delegating execution to the narrowest mature architectural entity.

# 2. Entity Classification Rules

## 2.1 Delegated Entity

### 2.1.1

A delegated entity is an architectural entity that carries a responsibility.

### 2.1.2

It may be:

1. a component,
2. a port,
3. a composed contract object,
4. a mediator,
5. a runtime actor.

### 2.1.3

A delegated entity is not necessarily a final code class.

---

## 2.2 Supporting Entity

### 2.2.1

A supporting entity participates in a responsibility without being the primary carrier.

### 2.2.2

Supporting entities may:

1. provide subordinate behavior,
2. provide composition material,
3. expose a related surface,
4. receive or pass a handoff,
5. maintain adjacent state.

---

## 2.3 Governed Object

### 2.3.1

A governed object is affected, carried, modified, admitted, composed, handed off, or traced by a responsibility.

### 2.3.2

A governed object is not automatically an actor.

### 2.3.3

Examples:

1. `Log Level Envelope`,
2. `Policy Object`,
3. `Queue Container`,
4. `Slot`,
5. `Envelope Slot Occupancy`,
6. `Delivery-Cycle Flag Set`,
7. `Dispatcher Feedback Report`.

---

## 2.4 Classification-Only Layer

### 2.4.1

A classification-only layer groups concepts for analysis.

### 2.4.2

It must not be used as a primary delegated entity.

### 2.4.3

Example:

```text id="s8dr2z"
Queue Runtime Layer = classification only
Queue Bundle Agent / Single Queue Moderator / Round Managers = actual delegated entities
```

The corrected matrix explicitly states that `Queue Runtime Layer` is only a classification layer and actual queue-runtime responsibility must be assigned to the concrete queue actors.

## 4.2 Definition

### 4.2.1

`Envelope Assembler` is a client-side composed contract object associated with the consuming entry path.

### 4.2.2

It is the correct delegation point for ingress preparation and envelope construction responsibilities.

### 4.2.3

It is not the entire Assembler Subsystem.

### 4.2.4

It is not the server-side receiving port.

---

## 4.3 Composition

### 4.3.1

`Envelope Assembler` composes:

1. `Validator`,
2. `Metadata Injector`,
3. `Timestamp Stabilizer`,
4. envelope assembly routine.

### 4.3.2

This composition reflects the corrected ingress flow:

```text id="iof9x4"
Log Level API
  contains / injects
    Envelope Assembler
      composes Validator, Metadata Injector, Timestamp Stabilizer
      produces Log Level Envelope
```

The corrected delegation matrix defines this ingress flow directly. 

---

## 4.4 Placement

### 4.4.1

`Envelope Assembler` is injected into, or composed inside, the `Log Level API`.

### 4.4.2

It belongs to the consuming ingress preparation side.

### 4.4.3

Its client-side placement does not make the whole Assembler Subsystem client-side.

---

## 4.5 Produced / Governed Artifact

### 4.5.1

Its main produced artifact is:


### 4.5.2

This envelope becomes the runtime carrier submitted to the Assembler side.
