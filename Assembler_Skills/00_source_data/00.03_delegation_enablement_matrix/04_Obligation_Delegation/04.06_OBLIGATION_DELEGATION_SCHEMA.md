# 2. Core Concepts

## 2.1 Responsibility

### 2.1.1

A responsibility is a canonical behavior, preservation duty, boundary rule, state-management duty, or architectural requirement.

### 2.1.2

Responsibilities are identified by:

```text id="pyuuyk"
RSP-CAN-###
```

## 2.2 Obligation

### 2.2.1

An obligation is the enforceable counterpart of a responsibility.

### 2.2.2

Obligations are identified by:

```text id="6w3uop"
OBL-CAN-###
```

### 2.2.3

Every top-level responsibility from `RSP-CAN-001` through `RSP-CAN-049` has one aligned top-level obligation from `OBL-CAN-001` through `OBL-CAN-049`.

## 2.3 Enforcing Entity

### 2.3.1

An enforcing entity is the entity responsible for ensuring that the obligation is upheld.

### 2.3.2

An enforcing entity may be:

1. an internal actor,
2. a boundary port,
3. a contract-side composite,
4. a mediator,
5. the accountable subsystem entity.

### 2.3.3

An enforcing entity must not be merely a governed object.

## 2.4 Supporting Entity

### 2.4.1

A supporting entity participates in obligation enforcement but is not the primary enforcement carrier.

### 2.4.2

Supporting entities may include:

1. external counterparties,
2. ports,
3. round managers,
4. queue actors,
5. optional future consumers.

## 2.5 Governed Object

### 2.5.1

A governed object is an object, state, artifact, carrier, directive, locator, boundary marker, or report controlled by an obligation.

### 2.5.2

Governed objects are not actors.

## 2.6 Sub-Responsibility

### 2.6.1

A sub-responsibility is a structured expansion of a top-level responsibility/obligation pair.

### 2.6.2

This package defines two sub-responsibility groups:

1. delivery-cycle flags under `RSP-CAN-025 / OBL-CAN-025`,
2. dispatcher feedback reports under `RSP-CAN-048 / OBL-CAN-048`.

## 2.7 Delivery-Cycle Flag

### 2.7.1

A delivery-cycle flag is a compact delivery-cycle state update.

### 2.7.2

Delivery-cycle flags are governed by:

```text id="85bjxa"
BCO-013 Delivery-Cycle Flag Set
```

### 2.7.3

Delivery-cycle flags are not detailed diagnostic reports.

## 2.8 Dispatcher Feedback Report

### 2.8.1

A dispatcher feedback report is a detailed diagnostic or reporting artifact.

### 2.8.2

Dispatcher feedback reports are governed by:

```text id="vl0nug"
BCO-014 Dispatcher Feedback Report
```

### 2.8.3

Dispatcher feedback reports are not delivery-cycle flags.

# 12. Anti-Collapse Validation Rules

## 12.1 Flag / Report Separation

### 12.1.1

Invalid:

```text id="kbl6ef"
Delivery-Cycle Flag Set = Dispatcher Feedback Report
```

### 12.1.2

Valid:

```text id="lk7rqu"
RSP-CAN-025 / OBL-CAN-025 → BCO-013 Delivery-Cycle Flag Set
RSP-CAN-048 / OBL-CAN-048 → BCO-014 Dispatcher Feedback Report
```

### 12.1.3
