# 3. Rollup Design Principles

## 3.1 No-New-Semantics Rule

### 3.1.1

The rollup package must not introduce new canonical architectural facts.

### 3.1.2

Any semantic statement in this package must be traceable to one or more upstream files.

### 3.1.3

If a rollup entry appears to introduce a new concept, it must be treated as invalid unless the concept is already registered in:

1. package `01_reference_markdown/`,
2. package `02_entity_enablement_indexes/`,
3. package `03_delegation_matrix/`,
4. package `04_obligation_delegation/`,
5. package `05_trace_coverage_registers/`.

---

## 3.2 Stable-ID Navigation Rule

### 3.2.1

All query paths must prefer canonical IDs over display names.

### 3.2.2

Display names are allowed for readability only.

### 3.2.3

Canonical IDs must remain the basis for:

1. joins,
2. validation,
3. traceability,
4. downstream relational keys,
5. graph node identity,
6. ontology seed extraction.

---

## 3.3 Source-Preservation Rule

### 3.3.1

Rollup rows must preserve source file lineage.

### 3.3.2

Every major rollup object should be able to answer:

```text id="u8on4h"
Which source file produced this fact?
Which source file validates this fact?
Which downstream file references this fact?
```

---

## 3.4 Anti-Collapse Preservation Rule

### 3.4.1

The rollup layer must preserve the release’s critical anti-collapse rules.

### 3.4.2

At minimum, this includes:

1. `Metadata Injector` is not a policy host.
2. `Queue Runtime Layer` is classification-only.
3. `Delivery-Cycle Flag Set` is not `Dispatcher Feedback Report`.
4. `Queue = Container`.
5. Waiting lists store `Queue Container Identifier`, not `Log Level Envelope`.
6. `Envelope Slot Occupancy` is binding/state only, not envelope lifecycle ownership.
7. `Registry / Delivery Port` is a contract surface, not a replacement for `Dispatcher`.
8. Typed/template contracts are not runtime COM interfaces.

---

## 3.5 Cross-Package Completeness Rule

### 3.5.1

The rollup package must be able to navigate across all canonical release packages.

### 3.5.2

At minimum, it must cover:

1. `00_release_manifest/`,
2. `01_reference_markdown/`,
3. `02_entity_enablement_indexes/`,
4. `03_delegation_matrix/`,
5. `04_obligation_delegation/`,
6. `05_trace_coverage_registers/`,
7. `06_rollup_indexes/`.

### 3.5.3

The optional archive package `07_packages/` is packaging-only and must not be treated as semantic authority.

6. Envelope Slot Occupancy is not lifecycle ownership.
7. Registry / Delivery Port does not replace Dispatcher.
8. Typed/template contracts are not runtime COM interfaces.

A valid rollup package must support:

1. relational derivation,
2. graph derivation,
3. query engine implementation,
4. CLI command design,
5. ontology seed extraction,
6. release validation reporting.
