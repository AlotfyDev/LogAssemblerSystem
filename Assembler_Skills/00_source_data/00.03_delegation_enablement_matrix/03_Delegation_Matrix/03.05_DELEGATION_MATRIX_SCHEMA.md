# 3. Core Concepts

## 3.1 Responsibility

### 3.1.1

A responsibility is a canonical system obligation of behavior, preservation, boundary control, or state management.

### 3.1.2

Responsibilities are identified by:

```text
RSP-CAN-###
```

### 3.1.3

This package recognizes 49 top-level responsibilities:

```text
RSP-CAN-001 ... RSP-CAN-049
```

## 3.2 Obligation

### 3.2.1

An obligation is the enforceable counterpart of a responsibility.

### 3.2.2

Obligations are identified by:

```text
OBL-CAN-###
```

### 3.2.3

Every responsibility row must map to at least one obligation ID.

## 3.3 Primary Delegated Entity

### 3.3.1

The primary delegated entity is the entity that carries responsibility execution or preservation.

### 3.3.2

Primary delegated entities must be mature architectural carriers, not merely classification labels.

## 3.4 Supporting Entity

### 3.4.1

A supporting entity participates in the responsibility but is not the main carrier.

### 3.4.2

Supporting entities may include:

1. external counterparties,
2. ports,
3. composed components,
4. other runtime actors,
5. optional future consumers.

## 3.5 Governed Object

### 3.5.1

A governed object is an object, carrier, artifact, locator, state, boundary marker, or directive affected by the responsibility.

### 3.5.2

Governed objects are not delegated actors unless explicitly promoted by later governance.

## 3.6 Enablement Family

### 3.6.1

An enablement family describes the capability surface required for a responsibility to be carried.

### 3.6.2

Enablement families are identified by:

```text
EN-FAM-###
```

## 3.7 Surface Type

### 3.7.1

A surface type classifies the functionality plane of the responsibility.

### 3.7.2

This package depends on the surface law:

```text
Administrative ≠ Managerial ≠ Consuming
```

### 3.7.3

Boundary, Contractual, and Observability surfaces may overlay those planes without erasing them.

RSP-CAN-001

```text
RSP-CAN-049
```

---

## 11.2 Primary Delegation

### 11.2.1

Every responsibility must have at least one primary delegated entity.

### 11.2.2

A primary delegated entity must be an actor, port, mediator, or accountable subsystem entity.

### 11.2.3

A governed object must not appear as a primary delegated entity.
