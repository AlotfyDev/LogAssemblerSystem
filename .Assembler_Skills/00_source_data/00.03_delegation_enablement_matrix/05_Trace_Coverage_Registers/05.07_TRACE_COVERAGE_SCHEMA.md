# 2. Core Concepts

## 2.1 Trace Coverage

### 2.1.1

Trace coverage means that a responsibility or obligation is not merely stated, but linked to its architectural evidence chain.

### 2.1.2

A complete trace chain may include:

1. responsibility,
2. obligation,
3. delegated entity,
4. supporting entity,
5. governed object,
6. enablement family,
7. surface,
8. contract,
9. gap closure,
10. doctrine,
11. taxonomy layer.

## 2.2 Gap Closure Coverage

### 2.2.1

A gap is not considered covered by name alone.

### 2.2.2

A gap closure is valid only when the register records:

1. gap ID,
2. closure statement,
3. risk prevented,
4. covering responsibilities,
5. preserving obligations,
6. delegated entities,
7. governed objects,
8. enablement families,
9. surfaces,
10. contract traces where applicable,
11. doctrine traces where applicable,
12. taxonomy traces where applicable,
13. anti-regression rule.

## 2.3 Contract Trace

### 2.3.1

A contract trace links a contract or contract-family row to the responsibilities and obligations derived from it.

### 2.3.2

Contract tracing preserves the derivation chain:

```text id="rr7y14"
Contract
→ Responsibility
→ Obligation
→ Entity
→ Governed Object
→ Enablement
```

## 2.4 Taxonomy Update

### 2.4.1

A taxonomy update is a record of a change, promotion, correction, split, or clarification introduced by the release.

### 2.4.2

Taxonomy updates are required when a concept becomes:

1. a new delegated entity,
2. a new governed artifact,
3. a corrected classification,
4. a policy-hosting exclusion,
5. a runtime terminology correction,
6. a surface split,
7. a contract-surface decomposition,
8. an ontology seed candidate.

## 2.5 New Entity Register

### 2.5.1

The new entities register records newly introduced or promoted responsibility carriers and governed artifacts.

### 2.5.2

This release registers:

1. `E-INT-011 Envelope Assembler`,
2. `E-INT-012 Assembler Ingress Port`,
3. `E-INT-013 Administrative / Ecosystem Port`,
4. `E-INT-014 Registry / Delivery Port`,
5. `BCO-012 Log Level Envelope`,
6. `BCO-013 Delivery-Cycle Flag Set`,
7. `BCO-014 Dispatcher Feedback Report`.

## 2.6 Ontology Seed Candidate

### 2.6.1

An ontology seed candidate is a traceable concept suitable for later conversion into an ontology class, relation, rule, or expert-skill knowledge unit.

### 2.6.2

This package does not finalize the ontology.

### 2.6.3

It prepares the traceable substrate for a future:

```text id="nr0l5e"
Assembler System Ontology Expert Skill
```
