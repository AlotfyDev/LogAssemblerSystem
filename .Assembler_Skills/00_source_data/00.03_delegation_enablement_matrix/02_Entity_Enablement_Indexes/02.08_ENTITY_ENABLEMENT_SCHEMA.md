
# 1. Package Scope

## 1.1

The `02_entity_enablement_indexes/` package defines queryable indexes for:

1. delegated entities,
2. supporting entities,
3. external counterparties,
4. classification-only layers,
5. enablement families,
6. surface types,
7. governed objects,
8. port and contract-side surfaces,
9. policy-consuming components,
10. entity-to-enablement joins.

## 1.2

The package does not define:

1. final class names,
2. final methods,
3. final source packages,
4. runtime algorithms,
5. persistence tables,
6. SQL indexes.

## 1.3

The package is a semantic and architectural indexing layer.

| `SURF-BOUNDARY`       | Boundary       | canonical base         |
| `SURF-CONTRACTUAL`    | Contractual    | canonical base         |
| `SURF-OBSERVABILITY`  | Observability  | canonical base         |

## 5.2 Count Rule

```text id="gzzefr"
canonical_base_surface_count = 6
derived_surface_classification_count = 1
```

## 5.3 Surface Separation Rule

### 5.3.1

The package preserves the Three Surface Law:

```text id="mrh5db"
Administrative ≠ Managerial ≠ Consuming
```

### 5.3.2

Boundary, Contractual, and Observability may overlay those surfaces, but they do not erase the base distinction.

# 6. CSV Schema Rules

## 6.1 General CSV Rules

### 6.1.1

All CSV files in this package must use a header row.

### 6.1.2

All canonical IDs must be represented as strings.

### 6.1.3

Multi-value fields must use semicolon-separated canonical IDs.

### 6.1.4

Natural-language fields must not be used as primary join keys.

### 6.1.5

Fields containing semicolon-separated values must preserve exact ID casing.

---

## 6.2 Required CSV Join Columns

### 6.2.1

At least one of the following canonical join columns must appear in every CSV file:

1. `entity_id`,
2. `governed_object_id`,
3. `enablement_family_id`,
4. `surface_id`,
5. `port_surface_id`,
6. `policy_component_row_id`.
